# Revia UI Design Guide

版本：v1.0

状态：MVP开发基准文档
1.设计理念
1.1产品定位
Revia的界面设计围绕“现代知识阅读工具（Modern Knowledge Workspace）”展开。
产品并非传统意义上的AI助手，也不是聊天机器人，而是一款帮助学生整理、阅读和管理考试资料的学习工具。因此，界面的重点不是突出AI，而是突出内容本身。
用户打开Revia的主要目的，是进入一个稳定、安静、能够长时间阅读的学习环境。因此，界面设计应尽可能降低干扰，让阅读体验始终保持在第一优先级。

1.2设计关键词
整个产品需要保持统一的设计语言，所有页面都围绕以下几个关键词展开：
安静（Calm）
阅读优先（Reading First）
清晰（Clear）
克制（Minimal）
长时间使用舒适（Comfortable）
界面应给用户一种“数字书桌”的感觉，而不是“AI正在与你聊天”。

2.整体风格
Revia的视觉风格融合两种产品气质：
一方面，它保留现代效率工具清晰、有秩序的信息组织方式，例如固定侧边栏、统一工具栏和稳定的页面结构；另一方面，它借鉴电子书阅读器和数字笔记工具的阅读体验，强调留白、文字层级和长时间浏览的舒适性。
最终形成一种介于Notion与阅读器之间，但更偏阅读体验的设计风格。
产品不追求科技感，也不追求教育App的活泼感，而是保持稳定、专业、耐看的视觉语言。

3.色彩规范
整体色彩采用低饱和、中性色方案。
背景颜色以暖白色为主，降低长时间阅读产生的视觉疲劳。正文采用深灰色而非纯黑色，提高阅读舒适度。
产品强调色采用低饱和灰绿色（或蓝绿色），统一用于：
当前选中的项目
当前目录
当前版本Tab
链接状态
Drawer标题
主要按钮
整个产品仅保留一种强调色，不使用多个品牌色竞争视觉焦点。
避免使用高饱和蓝色、紫色渐变、荧光色等容易产生科技产品印象的颜色。

4.字体规范
字体应优先保证阅读体验，而非品牌个性。
标题采用较高字重，用于建立信息层级；正文保持适中的字号和行高，支持长时间阅读；工具栏和按钮文字保持统一，不使用过小或过大的字体。
整体文字风格应偏书籍阅读，而非营销页面。

5.页面布局
5.1布局原则
学习页面采用固定三栏布局：
项目栏｜目录栏｜阅读区
页面宽度保持稳定，不因打开Drawer或切换功能发生明显变化。
整个产品始终围绕阅读区展开，阅读区是页面的视觉中心。

5.2项目栏
项目栏用于课程切换。
宽度保持较窄，仅承担项目管理功能。
折叠项目栏后，页面返回项目列表状态，不保留当前阅读内容。

5.3目录栏
目录栏用于当前项目内部导航。
目录采用树状结构展示章节和知识点。
点击目录后，正文平滑滚动至对应位置，而不是重新加载页面。
目录栏比项目栏略宽，因为用户在学习过程中会更频繁地使用目录导航。

5.4阅读区
阅读区承担所有主要内容展示。
整个页面一次只展示一个版本：
原文版
背诵版
关键词版
三个版本通过顶部Tab切换，不允许多个版本同时显示。
正文采用连续阅读模式，不使用知识卡片，不将知识点切分成大量独立模块。
阅读体验应更接近浏览一本整理好的教材，而不是浏览一个仪表盘。

6.Drawer设计规范
Drawer是Revia唯一的辅助操作区域。
Drawer从页面右侧滑出，并覆盖在正文上方，不改变页面布局。
关闭Drawer后，正文保持原有滚动位置，不发生位移。
Drawer在同一时间只承担一个任务，包括：
查看关键词对应的背诵内容。
单独编辑。
整体编辑。
禁止在同一个Drawer中同时展示多个互不相关的功能。
整体编辑模式下，三个版本采用折叠面板（Accordion）组织，用户按需展开，而不是同时展开所有内容。

7.工具栏规范
学习页面顶部工具栏保持简洁，仅保留高频操作。
工具栏包含：
Undo（撤销）
Redo（恢复）
Search（搜索）
Export（导出）
上传资料、项目管理等功能不应出现在学习页面工具栏中，而应分别放置在对应页面。

8.右键交互规范
Revia采用右键菜单作为主要内容管理入口。
用户右键点击不同对象时，应显示不同菜单：
点击项目：删除项目。
点击知识点：删除知识点。
点击要点：编辑、删除要点。
右键菜单仅在触发时显示，点击页面其他区域后立即关闭，不长期占据界面空间。

9.图标与控件
图标采用线性风格，保持统一视觉语言。
按钮避免过多填充色，默认采用文字按钮或轻量边框按钮，仅对主要操作使用强调色。
页面动画保持克制，仅用于：
Drawer打开与关闭。
页面滚动定位。
折叠面板展开。
下拉菜单展开。
避免使用复杂过渡动画或大面积动态效果。

10.禁止项（Design Constraints）
为了保证产品定位统一，设计过程中应避免以下情况：
不使用AI聊天窗口作为主要交互方式。
不设计首页Dashboard或数据大屏。
不采用大量知识卡片展示正文。
不在页面中加入拟人化AI角色。
不使用高科技风渐变背景。
不加入与学习无关的装饰元素。
不因功能增加而改变阅读区的布局。
所有新增功能都应优先思考是否会破坏“阅读优先”的原则。

11.Cursor UI开发约束
Cursor在开发页面时，应遵循以下规则：
**阅读内容始终是页面主体。**任何新增功能都不应压缩或遮挡阅读体验，除Drawer外，不新增其他悬浮操作区域。
**一个区域只负责一种任务。**项目栏负责项目管理，目录栏负责导航，阅读区负责展示内容，Drawer负责当前操作，避免一个区域承担多个职责。
**保持页面稳定。**打开Drawer、切换版本、编辑内容等操作不应导致整体布局发生变化，避免用户产生“页面跳动”的感觉。
**统一视觉语言。**所有新增组件应继承现有配色、圆角、字体和间距，不允许出现风格突变。
